# Strategies — 策略与战术

> 外链建设的宏观策略 + 具体战术。

## 策略一：SaaS 目录提交（最高 ROI）

### 为什么 SaaS 目录是最优解
- 获取成本：只填 1 个 URL + 基本信息
- Dofollow 率：接近 100%（"Visit website" 按钮默认 Dofollow）
- 存活持久：目录站一般不删已审核的 listing
- 2026 风险：大量目录站转付费，免费窗口正在关闭

### 执行方式
1. 搜索 "[niche] + submit tool/product/startup"
2. 找到有 "Submit" 入口的目录站
3. 填写产品名 + URL + 简短描述
4. 等审核（1-7 天）
5. 审核通过后验证 rel 属性

### 批量操作
一次准备品牌信息模板（名称、URL、描述、logo），然后批量提交到 20-50 个目录站。
ROI 远高于逐个写博客文章。

## 策略二：包管理器外链（Parasite SEO）

### 21 个包管理器经验
npm, PyPI, RubyGems, Packagist, Cargo, Go modules, Maven, NuGet 等。
每个包的页面都可以放 Homepage URL，大部分 Dofollow。

### 执行方式
1. 创建有意义的小型工具库（不是垃圾包）
2. 包的 Homepage 字段指向目标站
3. 发布到包管理器
4. 包管理器页面自动被 Google 索引

### 注意
不要发布垃圾/空包。Google 和包管理器社区都会惩罚。
包必须有实际功能，哪怕很小。

## 策略三：卫星站部署

### 概念
围绕主站部署多个小型独立站，每个站有自己的主题和内容，
但都通过自然链接指向主站。

### 平台选择
- Blogger（Google 生态，快速索引）
- WordPress.com（高 DR）
- dev.to（Dofollow 链接）
- Medium（nofollow 但品牌信号）
- velog.io（Dofollow）

### 注意事项
- 每个卫星站要有独立的内容主题
- 链接要自然，不要每个页面都链回主站
- 不同平台用不同邮箱注册
- 内容要有实际价值

## 策略四：dev.to 链接枢纽模式

### 概念
在 dev.to 发布多篇技术文章，文章之间互相链接，形成一个链接网络。
每篇文章都自然地包含主站链接。

### 执行方式
1. 规划 5-10 篇相关技术文章
2. 每篇文章 800-1500 字
3. 文章之间用"相关阅读"互相链接
4. 每篇包含 1-2 个主站链接（用有意义的锚文本）

### 优势
- dev.to 文章链接 Dofollow
- 多篇文章互相链接，权重传递
- Google 索引速度快
- 技术内容对 AI/SEO 工具天然适合

## 策略五：索引加速

### 新外链快速被 Google 收录的方法

1. **Ping 服务**
   ```
   https://www.google.com/ping?sitemap=你的页面URL
   ```

2. **Google Search Console**
   - 使用 URL 检查工具手动提交
   - 请求索引

3. **社交信号**
   - 在 Twitter/X 分享外链 URL
   - Google 爬取 Twitter 链接

4. **互相链接**
   - 已索引的页面链接到新外链
   - 形成爬取路径

5. **RSS Feed**
   - 如果平台支持 RSS，确保 URL 在 feed 中

## 竞品外链反查

### 正确姿势
1. 从 Ahrefs/Semrush 拿到竞品的精确 URL（具体到哪篇文章）
2. 去那篇文章检查竞品评论是否仍然存活
3. 存活 → 在同一篇文章下发评论（已证明这篇无审核）
4. 不存活 → 站长开始清理，整站跳过

### 选文章 Heuristics
- 评论数 > 50 且有明显 SEO 评论存活 = 站长不管这篇
- 老文章（>1 年）评论区更安全 = 站长不再关注
- 最近有新评论的老文章 = 评论区仍活跃且无审核

### 错误做法
从 Ahrefs 拿到竞品在 example.com 有外链 → 去 example.com 随便找一篇文章发评论 → 被审核/被删。
同一站的不同文章，审核策略不同。
